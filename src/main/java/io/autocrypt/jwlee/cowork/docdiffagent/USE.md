# DocDiffAgent 사용 가이드

`DocDiffAgent`는 기술 문서의 두 버전(Markdown 또는 AsciiDoc)을 비교하여 구조적 차이와 본문 내용의 기술적 변화를 분석합니다.

## 1. CLI 명령어 사용법

Spring Shell에서 `doc-diff` 명령어를 사용하여 분석을 실행합니다.

```bash
doc-diff --source-ver [소스버전] --source-path [소스경로] --target-ver [타겟버전] --target-path [타겟경로] [옵션]
```

### 파라미터 상세
- `--source-ver`: 비교의 기준이 되는 소스 버전 이름 (예: `0.3.4`)
- `--source-path`: 소스 문서 파일의 절대 또는 상대 경로
- `--target-ver`: 비교 대상이 되는 타겟 버전 이름 (예: `4.0.0`)
- `--target-path`: 타겟 문서 파일의 경로
- `-p` (Optional): 실행 시 LLM에 전달되는 프롬프트를 콘솔에 출력합니다.
- `-r` (Optional): 실행 시 LLM의 원본 응답을 콘솔에 출력합니다.

### 실행 예시
```bash
doc-diff --source-ver 0.3.4 --source-path 0.3.4.md --target-ver 4.0.0 --target-path 4.0.0.md
```

## 2. 분석 결과
분석이 완료되면 다음과 같은 형식의 한국어 마크다운 보고서가 출력됩니다:

1. **버전 개요**: 소스에서 타겟으로의 전반적인 변화 규모 요약.
2. **없어진 기능**: 타겟 버전에서 삭제된 섹션 및 기능 목록.
3. **추가된 기능**: 타겟 버전에서 새롭게 도입된 핵심 기능 분석.
4. **달라진 부분**: 매핑된 섹션 간의 기술적 차이점(어노테이션 변경, 속성 변화 등).
5. **마이그레이션 권고**: 하위 호환성 파괴 여부와 주요 대응 팁.

## 3. 작동 원리
1. **Map Phase**: `grep`을 통해 문서의 모든 헤더와 라인 번호를 추출하여 문서 지도를 만듭니다.
2. **Diff Phase**: LLM이 두 지도를 비교하여 추가/삭제/수정된 섹션을 인지합니다.
3. **Analyze Phase**: 변경이 감지된 섹션의 본문 라인 범위만 발췌하여 LLM이 기술적 차이를 심층 분석합니다.
4. **Synthesize Phase**: 모든 분석 조각을 모아 최종 보고서를 합성합니다.

## 4. 사전 요구 사항
- 분석 대상 문서 파일이 워크스페이스 내에 존재해야 합니다.
- `GEMINI_API_KEY` 환경 변수가 설정되어 있어야 합니다.
